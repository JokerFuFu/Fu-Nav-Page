# Fu 导航 · 场景模式 + 新手引导 v2 Product Spec

日期：2026-07-11 ｜ 状态：设计已获用户批准（四分叉已拍板） ｜ 执行：**codex**（三件套）

## 0. 决策记录

| 决策点 | 结论 |
|---|---|
| 模式 vs 工作区 | **合并升级**：工作区改名升级为「模式」，单一概念；旧数据一次性迁移 |
| 分组多属 | **允许**：一个分组可同时属于多个模式（数据反转为 mode 持有 groupIds） |
| 模式能力 | 分组集合 + **每模式小组件显隐 + 常用区开关**（用户选大方案）；壁纸/主题每模式配置本轮不做 |
| 引导升级 | 换**目标提升法**高亮（元素浮出遮罩+描边光圈），支持**弹窗内部引导**（步骤 before 钩子）；修「设置弹窗遮挡引导」bug |
| 执行 | codex 开发，Claude 主控 Playwright 全自动验收；判据合约照旧「只增不删、只翻 passes」 |

## 1. 问题

1. **场景切换名存实亡**：底部切换器对新用户只有「全部收藏/隐私模式」两项——「工作区」功能藏在分组右键里（先逐个分组设 page 才会出现在切换器），发现性差、配置繁琐；且一个分组只能属于一个工作区，「常用工具」没法同时出现在工作和学习场景。模式也只能过滤分组，无法控制小组件与常用区（学习模式想关掉娱乐常用区做不到）。
2. **新手引导有 bug 且能力不足**：设置里点「重看新手引导」，设置弹窗不关闭——遮罩（z=100）盖过弹窗（z=60）后，变暗的弹窗白卡糊在高亮目标前面，引导没法看。且单层遮罩挖洞的实现**天生无法引导弹窗内部元素**（如"云同步在这里"），引导覆盖不了设置内的关键功能。

## 2. 假设

把「模式」做成一等公民（一处管理、勾选即配、每模式定制界面），用户会真的按场景切换使用（而不是永远停在"全部"）；引导能进弹窗内部后，云同步/模式管理这类藏在设置里的高级能力会被新用户发现。机制：场景化让首屏信息密度匹配当下任务，引导决定高级功能的发现率。

## 3. 范围

### IN —— 两个工作包

**M · 场景模式系统**

1. **数据模型**（migrate 一次性迁移，禁常驻兜底）：
   ```js
   settings.modes = [{ id:'m<uid>', name:'工作',
                       groupIds:['g01',…],        // 该模式显示的分组（勾选式）
                       hiddenWidgets:['w-…',…],   // 该模式隐藏的小组件 id（默认 []=全显）
                       showFavs:true }]            // 该模式是否显示常用区（默认 true）
   settings.activeMode = null | 'privacy' | '<modeId>'   // null=全部
   ```
   迁移：对存量 `group.page` 的每个不同值建一个 mode（groupIds=含该 page 的分组）；`activePage!=='全部'`→activeMode=对应 modeId；`privacy===true`→activeMode='privacy'；迁移后 **delete** 每个 `g.page`、`settings.activePage`、`settings.privacy`（旧字段清除，避免双源）。
2. **渲染过滤**（layouts/fusion.js）：
   - `activeMode===null`：全显（现「全部收藏」行为）。
   - `'privacy'`：保持现隐私行为（只搜索/天气；进入时 active 归位 home 的 D5 修复保留）。
   - 模式 id：侧栏与主区只渲染 `mode.groupIds` 内的分组；首页小组件跳过 `hiddenWidgets` 中的 id；`showFavs===false` 时常用区整块不渲染。agent 派生卡（提醒/日报/磁盘）不按模式控制（沿用全局 hiddenAgentCards，范围注明）。
   - 命令面板/搜索**不受模式过滤**（全局直达，模式只影响首屏展示）。
3. **切换器**（底部 modeBtn 菜单）：全部收藏 / 自定义模式×N（勾选态）/ 隐私模式 / 分隔线 / **管理模式…**。当前模式高亮，切换即 save+rerender。modeBtn 图标：全部=layers、隐私=eye、自定义模式=layers（统一，不做每模式图标）。
4. **管理模式弹层** `openModeManager()`（openModal 子弹层，DESIGN.md 组件规范）：
   - 模式列表（名称+分组数），行内「编辑/删除」；「新建模式」按钮（promptModal 输名称）。
   - 编辑视图：名称输入、**分组勾选列表**（全部分组 checkbox）、**小组件勾选列表**（settings.widgets 逐个 toggle 显/隐）、**常用区开关** toggle。改动即写 mode 并 save。
   - 删除模式：无 confirm，走 `_offerUndo`（撤销恢复整个 mode 对象）；删除后若 activeMode 指向它→回 null。
   - 设置→高级 也放一个「管理模式」入口（与切换器菜单同一弹层）。
5. **分组右键菜单改造**：「移到工作区/新建工作区」改为「**所属模式**」子菜单——列出全部模式（勾选态，点击切换该分组在该模式的成员关系）+「管理模式…」。
6. **联动**：删除分组时从所有 mode.groupIds 移除该 id（删除撤销的快照需连带恢复受影响模式的 groupIds）；模式内分组全被删空时模式保留（空模式显示空态引导）。
7. README 工作区章节改写为「场景模式」。

**T · 新手引导 v2**

1. **bug 修复**：`startTour()` 开头统一 `core.closeModal()`（含从设置重看的场景），并清理可能残留的旧 tour DOM。
2. **高亮实现换目标提升法**（替换单层 box-shadow 挖洞）：
   - 全屏遮罩 `.fn-tour-mask`（z=100，scrim）；被引导元素加 `.fn-tour-hl`（`position:relative; z-index:110` + accent 描边光圈 + 呼吸动画，动画在 reduced-motion 守卫内）；
   - 目标在弹窗内时：其所在 `.fn-backdrop` 临时加 `.fn-tour-elevated`（z=105）——弹窗整体亮着浮出遮罩、目标再描边突出；
   - 步进/退出时移除全部临时 class；旧 `.fn-tour-spot` 实现删除。
3. **步骤引擎升级**：步骤结构 `{sel, title, text, before?, cleanup?}`——`before(core)` 异步钩子（如打开设置并展开某区，带元素出现轮询等待 ≤3s，超时跳过该步不卡死）；`cleanup(core)` 在离开该步时执行（如关闭 before 打开的弹层）。支持向前/向后步进时正确执行 cleanup/before 链。
4. **步骤表扩至 7 步**：① 侧栏分组 ② 搜索与 AI ③ 编辑锁 ④ 快捷入口 ⑤ **模式切换器**（新增，锚 `data-tour="mode"`）⑥ **设置内·同步与备份**（before：打开设置+展开该区，高亮区块；离开时关闭设置）⑦ 结尾卡（无锚点居中：完成语+「设置→高级可随时重看」）。
5. 视觉细节：步点进度（7 圆点）、气泡按目标位置四向智能翻转、Esc=跳过、窗口 resize 重定位——沿用并适配新引擎。
6. README 新手引导描述同步（7 步、可引导设置内部）。

### OUT
- 每模式壁纸/主题/搜索引擎配置；模式图标自定义；模式排序拖拽。
- agent 派生卡按模式显隐（沿用全局隐藏）。
- 引导的强制交互步（advanceOn:click 机制不做，全部「下一步」推进）。
- 快捷键切换模式。

### CUT（取舍）
- 模式数据放 `settings.modes`（mode 持有 groupIds），**不**在 group 上放 pages 数组——管理界面勾选式读写更直接，分组删除清理一处搞定。
- 空模式不自动删除（用户可能暂存）；「全部」「隐私」不可删不可改名（内置项）。

## 4. 用户体验（关键动线）

- 新用户：底部切换器点开 → 「管理模式…」→ 新建「学习」→ 勾选 3 个分组、隐藏硬件监控卡、关常用区 → 切到学习模式：首屏只剩学习相关分组+干净的小组件区。
- 老用户（有工作区数据）：升级后切换器里原工作区原样变成模式，无感迁移；右键分组现在是勾选式「所属模式」，一个分组可同时勾进工作+学习。
- 引导：从设置点「重看新手引导」→ 设置自动关闭 → 7 步引导；走到第 6 步设置弹窗自动打开、「同步与备份」区浮出遮罩被光圈圈住——用户第一次知道云同步在哪。

## 5. 验收标准（意图层，执行层判据见 eval）

1. 全新用户与存量工作区用户（构造 g.page 旧数据）迁移后行为正确，旧字段清除，无双源。
2. 模式的三层控制（分组/小组件/常用区）在切换时即时生效且持久；命令面板不受模式过滤。
3. 管理弹层全流程（建/改名/勾选/删除+撤销）可用，390 宽不破版。
4. 从设置重看引导不再被弹窗遮挡；第 6 步能正确高亮设置弹层内部区块；Esc 中途退出后无残留 class/弹层；reduced-motion 下可完整走完。
5. 全量回归（六界面×双主题）console 无红错；图标语言零违规；版本 3.22.0。

## 6. 成功指标（上线后观察）

- 自用与反馈中出现「切着模式用」的行为（activeMode 不再常年是 null）。
- 不再出现「引导被遮挡」「工作区在哪」类困惑。
